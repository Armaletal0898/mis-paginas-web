¿Qué es Linux y la Arquitectura del Kernel?

Rigurosamente hablando, **Linux** no es un sistema operativo completo, sino un **Kernel** (núcleo) libre, de código abierto y de tipo Unix-like, desarrollado inicialmente por Linus Torvalds en 1991.

El sistema operativo que utilizamos habitualmente se conoce de forma correcta como **GNU/Linux**, dado que combina el núcleo Linux con las herramientas, utilidades y compiladores del proyecto GNU necesarios para conformar un entorno operativo funcional.

El Rol del Kernel en la Infraestructura

El Kernel es el componente de software de más bajo nivel que actúa como la capa de abstracción intermedia entre el hardware físico de la máquina y las aplicaciones de usuario. Sus funciones críticas se dividen en:

  • Gestión de Memoria: Asignación y aislamiento del espacio de memoria RAM (User Space vs Kernel Space) para cada proceso.
  • Administración de CPU: Planificación de tareas (Scheduling) para determinar qué procesos acceden a los núcleos del procesador y por cuánto tiempo.
  • Controladores de Dispositivos (Drivers): Traducción de instrucciones genéricas de software a comandos de hardware específicos.
  • Llamadas al Sistema (System Calls): La interfaz con la que los programas solicitan servicios seguros al núcleo (ej: abrir un archivo, enviar un paquete de red).

Distribuciones de Linux (Distros)

Debido a la naturaleza modular y de código abierto del ecosistema, diferentes organizaciones y comunidades empaquetan el Kernel Linux junto con entornos gráficos, gestores de paquetes y configuraciones de software específicas para crear una Distribución.

Familias Principales de Distribuciones

  • Familia Debian: Enfocada en la estabilidad absoluta de la arquitectura. Utiliza el gestor de paquetes `.deb` (APT). Sus derivados más famosos incluyen a Ubuntu, Linux Mint, y entornos de seguridad y auditoría de redes como Parrot OS y Kali Linux.
  • Familia Red Hat (RHEL): El estándar corporativo y de servidores empresariales por excelencia. Utiliza el formato de paquetes `.rpm` (DNF). Sus variantes comunitarias e industriales directas incluyen a Rocky Linux, AlmaLinux y Fedora.
  • Familia Arch: Basada en el principio KISS (Keep It Simple, Stupid) y con un modelo de actualización continua (Rolling Release). Otorga un control total al usuario para construir el sistema desde cero. Su gestor nativo es Pacman.

Estructura del Sistema de Archivos (FHS)

A diferencia de los sistemas Windows que dividen la estructura en unidades físicas localizadas (C:, D:), Linux organiza todo bajo la estructura lógica del estándar Filesystem Hierarchy Standard (FHS), donde todo nace a partir de una única raíz denotada por una barra diagonal: /.

En Unix y Linux, se aplica la máxima de diseño: "En Linux, todo es un archivo" (los discos, los terminales, los procesos y las interfaces de red se representan y manipulan a través de archivos de texto o bloques especiales).

Directorios Críticos de la Raíz

  • /bin y /sbin: Contienen los archivos binarios ejecutables de los comandos esenciales del sistema (sbin está reservado para binarios de administración del superusuario root).
  • /etc: El corazón de la configuración del sistema. Aloja los archivos de configuración en formato de texto plano de todos los servicios locales y de red.
  • /home: Espacio personal de almacenamiento asignado a los usuarios del sistema (ej: `/home/jose`). El superusuario tiene su propio directorio aislado en /root.
  • /var: Almacena archivos de datos variables y dinámicos que cambian constantemente durante la ejecución, tales como bases de datos, colas de impresión y, críticamente, los **archivos de logs del sistema** (/var/log).
  • /dev: Contiene los archivos de dispositivos lógicos que mapean el hardware físico (ej: /dev/sda representa el primer disco duro de almacenamiento).
  • /proc y /sys: Sistemas de archivos virtuales basados en memoria RAM. No ocupan espacio en el disco duro; exponen directamente información del Kernel y de los procesos en tiempo real.

La Línea de Comandos (CLI) y Navegación Esencial

La terminal interactúa con el Kernel a través de un intérprete de comandos conocido como Shell (siendo Bash y Zsh los estándares más extendidos). Dominar la CLI requiere memorizar y combinar comandos fundamentales de navegación:

Comandos de Ubicación y Tránsito

  • pwd (Print Working Directory): Muestra la ruta absoluta del directorio actual en el que se encuentra posicionada la terminal.
  • cd (Change Directory): Cambia el directorio de trabajo actual. cd .. retrocede un nivel en la jerarquía, mientras que cd ~ lleva directamente al Home del usuario.
  • ls (List): Lista el contenido de un directorio. Sus parámetros avanzados más importantes son:
    • ls -l: Formato largo que detalla permisos, tamaños, propietarios y fechas de modificación.
    • ls -a: Muestra todos los archivos, incluyendo los archivos ocultos (aquellos cuyo nombre empieza por un punto, como `.bashrc`).

Manipulación y Visualización de Archivos

La administración efectiva del sistema se realiza modificando y gestionando ficheros directamente desde la consola mediante utilidades nativas de bajo nivel:

Comandos Operativos de Archivos

  • touch [archivo]: Crea un archivo de texto vacío o actualiza la marca de tiempo de un archivo existente.
  • mkdir [nombre]: Crea un nuevo directorio. El flag mkdir -p permite crear una ruta de directorios anidados de forma recursiva.
  • cp [origen] [destino]: Copia archivos. Para copiar directorios enteros junto con su contenido, se debe forzar el flag recursivo: cp -r.
  • mv [origen] [destino]: Mueve o renombra archivos y directorios dentro de la jerarquía de almacenamiento.
  • rm [archivo]: Elimina de forma definitiva archivos. Precaución: rm -rf elimina directorios y todo su contenido recursivamente de manera forzada sin pedir confirmación.

Comandos de Visualización

  • cat: Concatena y vuelca todo el contenido de un archivo directamente en la pantalla de la terminal.
  • less: Abre el archivo en un visor paginado interactivo que permite realizar búsquedas internas y desplazarse cómodamente con las flechas del teclado.
  • head -n 10 y tail -n 10: Muestran estrictamente las primeras o las últimas 10 líneas de un archivo respectivamente (útil para auditorías de logs en tiempo real mediante tail -f).

Filtros, Tuberías (Pipes) y Redirecciones

El poder real de la terminal de Linux radica en su capacidad para interconectar herramientas pequeñas y especializadas utilizando flujos de datos estándar.

Los Tres Canales Estándar

Cada proceso en Linux abre automáticamente tres canales de comunicación de datos identificados por descriptores numéricos:

  • Standard Input (stdin / 0): Flujo de entrada de datos, mapeado por defecto al teclado de la estación.
  • Standard Output (stdout / 1): Flujo de salida estándar para resultados exitosos, mapeado a la pantalla de la terminal.
  • Standard Error (stderr / 2): Flujo de salida independiente reservado exclusivamente para los mensajes de error del sistema.

Operadores de Redirección y Tuberías

  • >: Redirecciona el stdout de un comando hacia un archivo de texto, sobrescribiendo su contenido previo por completo.
  • >>: Redirecciona el stdout anexando la nueva información al final del archivo existente sin alterar los datos previos.
  • | (Tubería / Pipe): Toma el stdout del comando de la izquierda y lo inyecta directamente como el stdin del comando de la derecha, permitiendo encadenar procesamiento complejo.
  • grep [patrón]: Filtro de procesamiento de texto de alta eficiencia que extrae únicamente las líneas que coinciden con una expresión regular o patrón específico. Ejemplo de combinación: cat /var/log/auth.log | grep "Failed password" (lista intentos fallidos de inicio de sesión).

Gestión de Usuarios, Grupos y el Entorno Root

Linux es un sistema operativo inherentemente multiusuario y multitarea desde su concepción arquitectónica, diseñado para segregar de forma estricta los privilegios operativos.

Tipos de Usuarios y Ficheros del Sistema

  • Superusuario (root): La cuenta del administrador del sistema. Posee privilegios omnipotentes e irrestrictos sobre el Kernel, el hardware y los archivos del sistema. Su UID (User ID) es siempre 0.
  • Usuarios Regulares: Cuentas creadas para personal o servicios de aplicaciones con acceso restringido únicamente a su entorno local (Generalmente inician con UIDs a partir de 1000).
  • /etc/passwd: Archivo de texto plano que define las cuentas de usuario locales, detallando su nombre, UID, GID, directorio Home y el Shell por defecto.
  • /etc/shadow: Archivo seguro y altamente restringido que almacena los hashes criptográficos de las contraseñas cifradas de los usuarios.

Comandos de Gestión de Identidad

  • useradd y usermod: Comandos para crear nuevas identidades de usuario y modificar configuraciones de cuentas existentes (como asignación de grupos).
  • su [usuario]: Cambia temporalmente la sesión de la terminal a la cuenta de otro usuario.
  • sudo (Superuser Do): Permite a un usuario regular autorizado ejecutar un comando con los privilegios de seguridad del superusuario root sin comprometer la contraseña de administración central. Los usuarios permitidos se configuran en el archivo estructurado /etc/sudoers.

Permisos de Archivos Tradicionales (UGO)

La seguridad de los recursos en Linux se basa en un esquema rígido que asigna permisos específicos a tres clases de entidades distintas asociadas a cada archivo:

  • User (u): El propietario individual específico que creó o posee el archivo.
  • Group (g): El grupo de usuarios del sistema asignado al archivo que comparte los mismos permisos.
  • Others (o): Cualquier otro usuario del sistema que no es el propietario ni pertenece al grupo asignado.

Los Tres Tipos de Permiso Básicos

Cada clase de entidad puede tener una combinación de los siguientes tres permisos representados de forma simbólica o numérica (octal):

Permiso Símbolo Valor Octal Efecto en Archivos Efecto en Directorios
Read r 4 Leer o visualizar el contenido interno. Listar los nombres de archivos internos (ls).
Write w 2 Modificar, editar o guardar el archivo. Crear, eliminar o renombrar archivos dentro.
Execute x 1 Ejecutar el archivo si es un binario/script. Acceder o entrar al directorio (cd).

Comandos de Modificación

  • chmod [permisos] [archivo]: Cambia los permisos de acceso del archivo. Puede hacerse mediante notación octal (ej: chmod 755 script.sh otorga rwxr-xr-x) o simbólica (ej: chmod +x script.sh añade permiso de ejecución).
  • chown [propietario]:[grupo] [archivo]: Altera el propietario y/o el grupo asignado al archivo del sistema (operación restringida estrictamente a root).

Gestión de Procesos y Señales del Sistema

Un **proceso** es simplemente un programa ejecutable en memoria activa que consume recursos de CPU y RAM. Cada proceso es identificado unívocamente por el Kernel mediante un número entero llamado PID (Process ID).

Monitoreo y Control de Procesos

  • ps: Muestra una captura instantánea de los procesos activos. Su variante avanzada de administración es ps aux, que detalla el uso de recursos de todos los usuarios del sistema.
  • top / htop: Monitores interactivos en tiempo real que ordenan dinámicamente los procesos por consumo de hardware, temperatura y carga general del sistema.
  • Señales del Sistema: Mecanismos de comunicación asíncronos del Kernel para alterar la ejecución de un proceso mediante el comando kill:
    • kill -15 [PID] (SIGTERM): Señal de terminación suave y ordenada. Permite al proceso guardar configuraciones y cerrar archivos abiertos antes de morir.
    • kill -9 [PID] (SIGKILL): Terminación abrupta y forzada a nivel de Kernel. Destruye el proceso de inmediato sin importar su estado actual.

Gestión de Paquetes e Instalación de Software

En lugar de descargar instaladores ejecutables manuales de páginas web de terceros, las distribuciones modernas de Linux centralizan el software a través de servidores seguros conocidos como **Repositorios**.

Los **Gestores de Paquetes** de alto nivel automatizan la descarga, la verificación de firmas criptográficas de seguridad y la resolución automática de **dependencias** (librerías compartidas complementarias requeridas para que el programa funcione).

Ecosistemas de Gestión

  • Sistemas basados en Debian / Ubuntu (APT):
    • sudo apt update: Descarga e indexa las listas de versiones de paquetes actualizadas desde los repositorios.
    • sudo apt upgrade: Actualiza todos los paquetes e infraestructura instalados a su última versión segura.
    • sudo apt install [paquete]: Descarga e instala de forma limpia el software especificado.
  • Sistemas basados en Red Hat / RHEL (DNF): Utiliza comandos análogos estructurados bajo la sintaxis corporativa de paquetes RPM (ej: sudo dnf install nmap).

Configuración y Diagnóstico de Red Local

Linux cuenta con una pila de protocolos de red integrada directamente en el Kernel y se controla mediante utilidades del subsistema de red estándar:

Comandos Esenciales de Red

  • ip a (IP Address): Muestra el estado físico de todas las interfaces de red configuradas (ethernet, interfaces inalámbricas, loopback) junto con sus respectivas direcciones IP lógicas y máscaras de subred. Remplaza al comando obsoleto `ifconfig`.
  • ping [host]: Envía paquetes de control ICMP hacia una IP o dominio remoto para comprobar el estado de la conectividad básica de la red de extremo a extremo y medir la latencia de la línea de comunicación.
  • ss -tulpn: Herramienta avanzada de auditoría que lista detalladamente todos los sockets y puertos de red (TCP/UDP) abiertos en estado de escucha (Listening) local y detalla qué servicio o PID específico los está controlando. Reemplaza al comando heredado `netstat`.
