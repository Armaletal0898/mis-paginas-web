Arquitectura Interna de Windows (Kernel NT)

Los sistemas operativos modernos de Microsoft (Windows 10, 11 y Windows Server) se construyen sobre la evolución del diseño de la arquitectura **Windows NT**. Es un sistema de arquitectura híbrida enfocado en la modularidad y la robustez técnica.

Al igual que otros sistemas avanzados, divide de manera rígida la ejecución del software en dos modos de operación lógicos para garantizar la protección del hardware:

Anatomía de Modos de Ejecución

  • Modo Usuario (User Mode): Las aplicaciones comunes (navegadores, editores de código, malware de espacio de usuario) se ejecutan aquí. No tienen acceso directo al hardware físico ni a la memoria central. Si una aplicación falla en User Mode, se cierra de forma aislada sin comprometer el sistema completo.
  • Modo Kernel (Kernel Mode): Reservado para el núcleo central del sistema operativo, el subsistema gráfico ejecutivo y los controladores de bajo nivel (Drivers). El código en Kernel Mode tiene privilegios irrestrictos sobre la CPU y la memoria. Un fallo o vulnerabilidad explotada en este modo provoca un colapso crítico del sistema, conocido como **BSOD (Blue Screen of Death)**.

El Registro de Windows

El **Registro de Windows** es una base de datos jerárquica centralizada fundamental donde el sistema operativo guarda todas las configuraciones de hardware, opciones del software de terceros, controladores de dispositivos, preferencias de usuario y parámetros de seguridad críticos.

Es una pieza clave en auditorías y ciberseguridad, ya que los atacantes lo utilizan frecuentemente para inyectar técnicas de **Persistencia** (para arrancar malware de forma invisible junto con el sistema operativo).

Las Cinco Colmenas Principales (Hives)

  • HKEY_CLASSES_ROOT (HKCR): Gestiona las asociaciones de extensiones de archivos (ej: qué programa abre un archivo `.pdf`) y componentes OLE/COM.
  • HKEY_CURRENT_USER (HKCU): Almacena la configuración específica del perfil del usuario que mantiene la sesión activa en ese instante (fondos de pantalla, configuraciones locales).
  • HKEY_LOCAL_MACHINE (HKLM): La colmena más crítica. Contiene las configuraciones globales de hardware y del sistema operativo aplicables a todos los usuarios de la máquina (subllaves `SYSTEM`, `SAM`, `SECURITY`, `SOFTWARE`).
  • HKEY_USERS (HKU): Aloja de forma individualizada los perfiles de todos los usuarios configurados localmente en la máquina.
  • HKEY_CURRENT_CONFIG (HKCC): Mapea datos temporales sobre el perfil de hardware utilizado por el sistema durante el arranque actual.

Sistemas de Archivos y Estructura de Directorios

Windows organiza el almacenamiento físico dividiéndolo en volúmenes lógicos denotados por letras (C:, D:). El estándar industrial nativo para discos del sistema es **NTFS (New Technology File System)**, el cual ofrece capacidades avanzadas de seguridad corporativa en comparación con el antiguo formato FAT32.

Características Avanzadas de NTFS

  • Listas de Control de Acceso (ACLs): Permiten definir permisos hiperespecíficos por usuario o por grupos de Active Directory para cada archivo o carpeta.
  • Cifrado EFS (Encrypting File System): Cifrado nativo a nivel de archivo basado en la cuenta de usuario.
  • Journaling (Transaccional): Registra los cambios de archivos antes de escribirlos para evitar la corrupción de datos en caso de un apagón repentino.
  • Alternate Data Streams (ADS): Característica que permite bifurcar metadatos ocultos dentro de un mismo archivo (frecuentemente explotado por ciberatacantes para ocultar código malicioso dentro de imágenes o archivos de texto legítimos).

Directorios Críticos del Sistema

  • C:\Windows\System32: El directorio más sagrado. Contiene las librerías dinámicas del sistema (.dll), los controladores del Kernel y las herramientas ejecutables del sistema (cmd.exe, taskmgr.exe).
  • C:\Users\[Usuario]\AppData: Carpeta oculta donde las aplicaciones guardan configuraciones locales. Se divide en Local, LocalLow y Roaming.
  • C:\Windows\SysWOW64: Reservado en arquitecturas de 64 bits para almacenar los binarios y librerías compatibles con aplicaciones heredadas de 32 bits.

Consola de Comandos Tradicional (CMD / Command Prompt)

Heredada directamente de la arquitectura de MS-DOS, el **CMD** es el intérprete de comandos clásico de Windows. Aunque ha sido superado por PowerShell, sigue siendo vital en entornos corporativos para administración básica y scripting automatizado `.bat` de bajo impacto.

Comandos Esenciales de CMD

  • dir: Lista el contenido del directorio actual (equivalente a `ls` en Linux).
  • cd [ruta]: Cambia de directorio. A diferencia de Linux, para cambiar de unidad física se debe escribir la letra directamente (ej: D:).
  • type [archivo]: Muestra el contenido de un archivo de texto directamente en la terminal (equivalente a `cat` en Linux).
  • mkdir [nombre] y rmdir /s /q [nombre]: Creación de directorios y eliminación forzada/silenciosa de árboles de carpetas completos.
  • copy y move: Duplica o traslada archivos dentro del almacenamiento local.

PowerShell Avanzado y Automatización

**PowerShell** es un motor de automatización de tareas y un marco de scripting de última generación muy superior al CMD tradicional. A diferencia de la terminal de Linux o CMD (que procesan puro texto plano), PowerShell se basa en la programación **Orientada a Objetos** construida sobre la arquitectura **.NET** de Microsoft.

Esto significa que el output de un comando no es texto, sino un objeto estructurado que contiene propiedades, métodos y tipados de datos manipulables.

Estructura Cmdlet (Verbo-Sustantivo)

Los comandos nativos de PowerShell se denominan **Cmdlets** y siguen una sintaxis estricta y predecible de un verbo de acción y un sustantivo operativo:

  • Get-Service: Obtiene el listado de todos los servicios del sistema y su estado actual.
  • Start-Service -Name "wuauserv": Inicia un servicio específico (en este ejemplo, Windows Update).
  • Get-Process | Where-Object {$_.CPU -gt 50}: Captura los procesos en vivo y, mediante una **Tubería de Objetos**, filtra en tiempo real aquellos cuyo consumo de CPU supere el 50%.
  • Get-Content [archivo]: Lee y extrae el contenido estructurado de un archivo de texto o log.

Mecanismos de Autenticación, Cuentas y Active Directory

Windows maneja de forma diferenciada la seguridad y autenticación dependiendo de si el entorno es una máquina aislada o una red corporativa masiva.

Autenticación Local: La Base de Datos SAM

En estaciones locales, las credenciales de usuarios no se guardan en texto plano, sino cifradas dentro del archivo **SAM (Security Accounts Manager)** ubicado de forma protegida en C:\Windows\System32\config\SAM.

  • La SAM está bloqueada por el Kernel en tiempo de ejecución. Almacena hashes criptográficos conocidos como **NTLM hashes** para validar la identidad de los usuarios locales.
  • Administrador Local (Administrator): La cuenta con control absoluto local de la máquina. Su identificador de seguridad (SID) termina siempre de forma fija en la extensión relativa **-500**.

Autenticación Corporativa: Active Directory (AD) y Kerberos

En redes empresariales, la seguridad local se centraliza mediante **Active Directory**. Las cuentas se administran desde un servidor centralizado conocido como **Controlador de Dominio (Domain Controller)** y el protocolo estándar para la autenticación segura ya no es NTLM, sino **Kerberos** (basado en un sistema de tickets criptográficos de tiempo confidencial).

Permisos NTFS y Gestión mediante ICACLS

En Windows, los permisos sobre archivos y carpetas NTFS se evalúan mediante estructuras complejas conocidas como **DACL (Discretionary Access Control List)**, compuestas por múltiples entradas llamadas **ACE (Access Control Entries)**.

Tipos Principales de Permisos NTFS

  • Full Control (Control Total): Permite leer, escribir, modificar, ejecutar, borrar el archivo y, críticamente, cambiar los permisos de seguridad y tomar posesión (Ownership) del elemento.
  • Modify (Modificar): Permite ver y editar los datos internos, además de eliminar el archivo, pero prohíbe la alteración de las directivas de seguridad.
  • Read & Execute / Read / Write: Permisos segmentados para la visualización de datos, inyección de nuevas líneas y ejecución de binarios.

Herramienta Administrativa Avanzada: ICACLS

Para auditar o modificar los permisos NTFS de forma masiva desde la consola corporativa (CMD o PowerShell), se utiliza el comando **ICACLS**:

  • icacls "C:\Compartido": Muestra de forma detallada las cuentas de usuarios y grupos que tienen acceso a la carpeta junto con sus respectivos flags de herencia.
  • icacls "C:\Archivo.txt" /grant Jose:F: Otorga permiso de control total (F = Full Control) explícito al usuario local "Jose".
  • icacls "C:\Archivo.txt" /deny Invitado:M: Deniega de forma explícita el permiso de modificación (M = Modify) a la cuenta de Invitados. Nota técnica de Windows: Las denegaciones explícitas siempre ganan prioridad sobre los permisos otorgados por herencia.

Gestión de Procesos, Servicios y el Proceso lsass.exe

En Windows, un proceso es una instancia en ejecución que cuenta con su propio espacio virtual de memoria direccionada. Los procesos en segundo plano que no interactúan con el escritorio gráfico del usuario se conocen como **Servicios** (equivalentes a los *daemons* de Linux).

Procesos del Sistema Críticos (Core Processes)

Identificar anomalías en los procesos nativos es crucial para detectar intrusiones de ciberseguridad avanzada. Los procesos legítimos obligatorios del sistema incluyen:

  • system: Proceso de Kernel que corre en un PID fijo (generalmente PID 4). Ejecuta hilos en modo Kernel.
  • smss.exe (Session Manager Subsystem): El primer proceso en modo usuario. Se encarga de iniciar las sesiones del sistema operativo.
  • svchost.exe (Service Host Process): Un proceso contenedor genérico que Windows utiliza para ejecutar múltiples servicios del sistema desde librerías dinámicas (DLLs). Es normal ver docenas de instancias de `svchost.exe` corriendo en paralelo.
  • lsass.exe (Local Security Authority Subsystem Service): El proceso encargado de verificar las contraseñas de los usuarios, generar tokens de acceso y aplicar las políticas de seguridad locales. Alerta de Ciberseguridad: Los atacantes avanzados intentan volcar la memoria de este proceso (`lsass.exe`) mediante herramientas como Mimikatz para extraer hashes NTLM o contraseñas en texto plano residentes en caché.

Visor de Eventos de Windows (Event Viewer)

Windows registra de forma exhaustiva la actividad del sistema, errores de aplicaciones, advertencias de hardware e inicios de sesión a través de los **Event Logs**. Estos se almacenan internamente en archivos estructurados con extensión binaria .evtx.

Los Tres Canales Principales de Eventos

  • Application (Aplicación): Registra eventos, fallas catastróficas y reportes generados por el software instalado por terceros o componentes del framework local.
  • System (Sistema): Almacena mensajes emitidos directamente por el Kernel NT de Windows, controladores de hardware, fallos de red globales y alertas de parches de actualización.
  • Security (Seguridad): El canal más importante para auditoría forense informática. Registra los intentos de inicio de sesión (exitosos y fallidos), cambios en los privilegios de los usuarios y auditorías de acceso a objetos del sistema de archivos.

Identificadores de Eventos Críticos (Event IDs)

En auditorías de seguridad, existen identificadores numéricos universales (**Event IDs**) que un especialista de TI debe monitorizar de inmediato para cazar intrusiones:

Event ID Significado Técnico Contexto de Monitoreo
4624 Inicio de sesión exitoso (Anclado al tipo de inicio: interactivo, red, servicio). Comprobar quién y desde qué IP remota ingresó a la máquina.
4625 Intento de inicio de sesión fallido. Vital para detectar ataques de fuerza bruta automatizados en tiempo real.
4720 Una nueva cuenta de usuario local fue creada en el sistema. Alerta máxima ante la posible creación de un usuario backdoor por persistencia de malware.
4672 Inicios de sesión con privilegios especiales (Asignación de derechos de Administrador). Rastrea la elevación de privilegios de cuentas comprometidas.
