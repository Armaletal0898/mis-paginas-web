Lab 1: Despliegue de un Homelab de Pentesting Segura

Objetivo: Diseñar un entorno aislado utilizando hipervisores (Type-1 como Proxmox VE o Type-2 como VirtualBox/VMware) para ejecutar pruebas de penetración sin exponer la red local doméstica o corporativa.

Arquitectura del Laboratorio Dedicado

  • Máquina Atacante: Distribución especializada (Parrot OS o Kali Linux) configurada en modo puente interno o red nativa del laboratorio.
  • Máquina Víctima 1 (Linux): Metasploitable 3 o contenedores LXC vulnerables expuestos deliberadamente.
  • Máquina Víctima 2 (Windows): Instancia de Windows Server o Windows 10 con defensas controladas para análisis del Registro y payloads.
  • Segmentación de Red: Configurar el switch virtual del hipervisor en modo **Host-Only** o **Internal Network** para evitar fugas de tráfico (*Traffic Bleeding*).

Playbook de Configuración de Red en Red Interna (CLI / VirtualBox)

Si manejas configuraciones rápidas por comandos, puedes forzar las tarjetas de red de tus máquinas virtuales a una red aislada llamada SecNetLabs usando la terminal:

VBoxManage modifyvm "Kali-Atacante" --nic1 intnet --intnet1 "SecNetLabs"
VBoxManage modifyvm "Windows-Victima" --nic1 intnet --intnet1 "SecNetLabs"
⚠️ Alerta de Seguridad Crítica: Nunca configures la máquina víctima en modo Bridged (Puente) directo a tu router principal. El software vulnerable o malware en ejecución podría escanear e infectar dispositivos reales de tu hogar o empresa.

Lab 2: Segmentación de Redes mediante VLANs y Enrutamiento Inter-VLAN

Objetivo: Mitigar los vectores de ataque por difusión (Broadcast) dividiendo de forma lógica una infraestructura física en subredes aisladas e independientes para el tráfico de Datos, Voz y Gestión.

Escenario de Red Corporativa

Nombre de VLAN ID de VLAN Segmento IP de Red Propósito Técnico
VLAN_ADMIN 10 192.168.10.0/24 Interfaces de gestión (SSH, Switches, Proxmox, PDUs).
VLAN_USER 20 192.168.20.0/24 Estaciones de trabajo de empleados comunes.
VLAN_DMZ 30 192.168.30.0/24 Servidores web públicos y APIs expuestas a internet.

Comandos de Implementación en Switch Cisco (IOS)

Configuración del enlace troncal (*Trunk*) hacia el Router (Router-on-a-Stick) y asignación de puertos específicos:

! Creación de las VLANs en la base de datos local
Switch# configure terminal
Switch(config)# vlan 10
Switch(config-vlan)# name VLAN_ADMIN
Switch(config-vlan)# vlan 30
Switch(config-vlan)# name VLAN_DMZ

! Configuración de puerto de acceso para un servidor en la DMZ
Switch(config)# interface fastEthernet 0/5
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 30
Switch(config-if)# no shutdown

! Configuración del puerto Troncal hacia el Router/Firewall
Switch(config)# interface gigabitEthernet 0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,30

Lab 3: Análisis Forense de Tráfico y Sniffing con Wireshark

Objetivo: Capturar frames de red en tiempo real para interceptar fugas de datos confidenciales transmitidos a través de protocolos heredados no cifrados.

Filtros de Visualización Esenciales (BPA - Best Practice)

Para depurar capturas de red masivas (`.pcap`), los filtros de Wireshark aíslan anomalías de forma inmediata:

  • http.request.method == "POST": Filtra las peticiones web de envío de datos. Útil para identificar credenciales expuestas en texto plano.
  • tcp.flags.syn == 1 && tcp.flags.ack == 0: Muestra intentos de inicio de conexión TCP. Si hay miles desde una sola IP externa, indica un escaneo SYN o ataque DoS.
  • dns.flags.response.rcode != 0: Filtra respuestas DNS que devolvieron errores. Crucial para detectar técnicas de *DNS Tunneling* o resolución errática de malware.

Flujo Práctico: Extracción de Archivos de una Captura

  1. Inicia la captura en la interfaz activa de tu laboratorio (ej: eth0 o wlan0).
  2. Ejecuta un flujo de descarga no cifrado en la máquina víctima.
  3. Detén la captura en Wireshark y navega al menú superior: File -> Export Objects -> HTTP...
  4. Elige el archivo interceptado de la lista y guárdalo localmente en tu sistema operativo para analizar su firma MD5/SHA256.

Lab 4: Fortalecimiento de Sistemas (Hardening) en Linux/Server

Objetivo: Reducir la superficie de exposición de un servidor de producción Linux mediante la restricción de privilegios, auditoría de software y securización del demonio SSH.

Script de Automatización de Directivas de Seguridad

Ejecuta este bloque de comandos con privilegios elevados (`sudo`) para auditar puertos abiertos y robustecer la configuración del archivo central /etc/passwd y las llaves del sistema:

# 1. Comprobar servicios escuchando en sockets de red activos
ss -tulpn

# 2. Deshabilitar el inicio de sesión directo del superusuario root por SSH
sudo sed -i 's/#PermitRootLogin prohibit-password/PermitRootLogin no/' /etc/ssh/sshd_config
sudo sed -i 's/PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config

# 3. Restringir los métodos de autenticación a llaves criptográficas públicas únicamente
sudo echo "PasswordAuthentication no" >> /etc/ssh/sshd_config

# 4. Reiniciar el servicio para aplicar los cambios de inmediato
sudo systemctl restart sshd

Lab 5: Análisis del Registro de Windows y Persistencia de Malware

Objetivo: Identificar e inyectar configuraciones en llaves del Registro de Windows utilizadas comúnmente por amenazas persistentes avanzadas (APTs) para ejecutarse de manera automática tras un reinicio del sistema.

Inyección Manual de Persistencia (Simulación de Amenaza)

Desde una terminal de PowerShell ejecutada como Administrador, corre el siguiente cmdlet para forzar la apertura automática de un ejecutable de pruebas junto con el inicio del sistema operativo:

New-ItemProperty -Path "HKLM:\Software\Microsoft\Windows\CurrentVersion\Run" -Name "UnderCodeBackdoor" -Value "C:\Windows\System32\calc.exe" -PropertyType String -Force

Análisis e Inspección Forense

Para cazar este tipo de persistencias en una auditoría de TI real, inspecciona la ruta utilizando el comando nativo u observa los cambios de inmediato desde la herramienta gráfica regedit.exe:

Get-ItemProperty -Path "HKLM:\Software\Microsoft\Windows\CurrentVersion\Run"

Lab 6: Escaneo de Vulnerabilidades y Auditoría de Puertos con Nmap

Objetivo: Identificar puertos abiertos, deducir de forma pasiva/activa versiones de servicios de red corriendo y mapear vulnerabilidades conocidas (CVEs) asociadas a la máquina objetivo.

Comandos Tácticos de Nmap para Auditoría

  • Escaneo Sigiloso SYN (Stealth Scan): `nmap -sS -Pn 192.168.33.10` (Evita completar el saludo de tres vías TCP para evadir logs básicos).
  • Detección Agresiva de Versiones y OS: `nmap -sV -sC -O -T4 192.168.33.10` (Ejecuta scripts por defecto, extrae banners del sistema de archivos y mapea el sistema operativo).
  • Auditoría Automatizada de Vulnerabilidades: `nmap --script vuln 192.168.33.10` (Utiliza el motor de scripting NSE para comprobar de forma segura la exposición a fallos críticos como EternalBlue o Log4j).

Lab 7: Monitoreo Centralizado y Detección de Amenazas con Wazuh SIEM

Objetivo: Instalar y configurar un agente de monitoreo de seguridad en un endpoint para centralizar logs, auditar la integridad de archivos (FIM) y detectar ataques de fuerza bruta en tiempo real.

Flujo de Despliegue Técnico

Wazuh opera bajo una arquitectura Servidor-Agente. Cuando un evento anómalo se dispara en el cliente, el agente cifra el log y lo transmite al manager para su correlación con reglas de seguridad.

Fases del Laboratorio SIEM

  1. Desplegar el Wazuh Manager en una instancia dedicada (ej: contenedor Proxmox con Ubuntu Server).
  2. Instalar el agente en la máquina que deseas auditar. En Windows, puedes desplegarlo mediante PowerShell con un único comando automatizado provisto por el panel.
  3. Configurar el módulo FIM (File Integrity Monitoring) en el archivo ossec.conf para vigilar directorios altamente críticos:
<syscheck>
  <directories check_all="yes" realtime="yes">/etc/passwd,/etc/shadow</directories>
</syscheck>

Cualquier modificación o adición de líneas en esos ficheros generará una alerta inmediata de Severidad Alta (Nivel 7+) en tu dashboard centralizado de Wazuh.
